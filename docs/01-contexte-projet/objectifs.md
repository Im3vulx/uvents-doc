# Présentation du projet

## Résumé

**ONY** est une **Progressive Web App (PWA)** conçue pour faciliter la découverte et la promotion d’événements locaux.
L’application vise à centraliser des événements de proximité dans une interface claire, mobile-first et orientée usage réel, avec une logique de consultation rapide via une **carte interactive**, des **filtres personnalisés** et des **parcours simplifiés**.

Le projet s’inscrit dans une démarche de construction d’un **MVP fonctionnel**, avec une attention particulière portée à :

- la cohérence du parcours utilisateur ;
- la simplicité d’usage ;
- la scalabilité progressive ;
- la maîtrise des coûts techniques ;
- la maintenabilité du code et de l’infrastructure.

## Problème adressé

Le projet part d’un constat simple : les grandes plateformes existantes couvrent souvent des segments spécifiques de l’événementiel, mais répondent moins bien au besoin de **découverte locale simple, rapide et contextualisée**.

L’ambition d’ONY est donc de proposer une application :

- plus proche du terrain ;
- centrée sur la vie locale ;
- adaptée à des usages mobiles ;
- capable de servir à la fois les utilisateurs finaux et les organisateurs.

## Positionnement du produit

ONY adopte une approche **map-first** et **mobile-first**.

La carte interactive constitue le cœur de l’expérience :

- elle permet de visualiser rapidement les événements autour de soi ;
- elle rend la découverte plus intuitive qu’une simple liste ;
- elle aligne le produit avec sa promesse de proximité.

Le projet cherche à se différencier de plateformes plus génériques par :

- une expérience plus directe ;
- une interface plus légère ;
- une relation plus forte au territoire ;
- un parcours de découverte plus naturel.

## Publics visés

Le projet s’adresse à plusieurs profils :

### Utilisateurs

Les utilisateurs recherchent :

- des sorties proches ;
- des événements cohérents avec leurs goûts ;
- une navigation simple et rapide ;
- des parcours courts entre découverte et action.

### Organisateurs

Les organisateurs recherchent :

- un moyen simple de publier un événement ;
- une meilleure visibilité locale ;
- une logique de gestion plus accessible que certains outils concurrents ;
- un outil exploitable sur mobile.

## Origine du nom ONY

Le projet a initialement porté le nom **Uvents**.
Le changement vers **ONY** répond à une volonté de sortir d’un nom trop descriptif, peu différenciant et trop proche du vocabulaire courant de l’événementiel.

Le nom ONY a été retenu pour :

- sa brièveté ;
- sa force visuelle ;
- sa capacité à construire une marque plus mémorable ;
- sa résonance implicite avec l’idée de mouvement et d’action.

## Nature du projet

Le projet combine plusieurs dimensions :

- **produit numérique** : création d’une PWA complète ;
- **projet technique** : architecture frontend, backend logique, base de données, CI/CD, monitoring ;
- **projet documentaire** : besoin d’une documentation maintenable et à jour ;
- **projet évolutif** : la base posée aujourd’hui doit permettre des extensions futures sans refonte complète.

## État actuel

À ce stade, ONY dispose déjà d’un socle conséquent :

- authentification utilisateur ;
- découverte d’événements ;
- carte interactive ;
- pages détail ;
- préférences et filtres ;
- billetterie simulée ;
- scan de billets ;
- parcours organisateur en cours de structuration ;
- infrastructure de déploiement et supervision déjà en place ;
- harmonisation UI/UX front largement avancée.

Le projet n’est pas encore considéré comme totalement finalisé, mais il a dépassé le stade du prototype brut.
Il repose désormais sur une base suffisamment solide pour être documentée de manière durable.

## Ambition documentaire

Cette documentation ne sert pas uniquement à répondre à une exigence académique.Elle doit devenir un outil opérationnel permettant :

- de comprendre rapidement le projet ;
- de retrouver les décisions structurantes ;
- de documenter les flux, composants et infrastructures ;
- de simplifier la maintenance et la reprise du projet.
