# Démarche continue

## Objectif de cette section

Cette page présente la **démarche continue** d’amélioration de l’accessibilité et de la qualité dans **ONY**.

L’objectif est d’expliquer comment ces sujets doivent être traités dans le temps, de manière réaliste, progressive et intégrée au projet.

## Principe général

L’accessibilité et la qualité ne se règlent pas une fois pour toutes dans un projet en évolution.

Elles doivent être entretenues dans le temps, au fil :

- des nouveaux écrans ;
- des corrections ;
- des évolutions UX ;
- des refontes ;
- des validations produit.

La bonne approche consiste donc à installer une logique d’amélioration continue plutôt qu’une promesse ponctuelle sans suivi.

## Pourquoi une démarche continue est nécessaire

Sans démarche continue, plusieurs effets apparaissent rapidement :

- retour des incohérences ;
- dégradation progressive de l’interface ;
- dette UX ;
- écarts entre écrans anciens et récents ;
- oubli des bonnes pratiques d’accessibilité.

Une amélioration isolée non suivie ne suffit pas à maintenir un bon niveau dans la durée.

## Logique d’amélioration progressive

Pour ONY, la démarche la plus pertinente consiste à avancer par étapes.

Cela peut passer par :

- l’identification des points les plus visibles ;
- la correction des problèmes les plus gênants ;
- l’harmonisation des composants récurrents ;
- l’intégration de critères simples dans les validations ;
- la documentation des décisions prises.

Cette logique est compatible avec un projet en mouvement.

## Intégration dans le cycle projet

L’accessibilité et la qualité ne doivent pas être traitées uniquement à la fin.

Elles doivent être prises en compte :

- lors de la conception ;
- lors du développement ;
- lors de la relecture ;
- lors des tests ;
- lors de la recette.

Plus elles sont intégrées tôt, moins les corrections deviennent coûteuses ensuite.

## Points d’appui concrets

Une démarche continue peut s’appuyer sur plusieurs leviers concrets :

- conventions UI partagées ;
- points de contrôle simples en recette ;
- revue visuelle des écrans importants ;
- attention particulière aux composants partagés ;
- documentation des améliorations faites et à faire.

L’objectif est de rendre ces sujets opérationnels, et non purement théoriques.

## Posture recommandée

La posture la plus crédible pour ONY est la suivante :

- reconnaître l’état réel du projet ;
- améliorer progressivement sans surpromettre ;
- intégrer les bons réflexes dans le workflow ;
- traiter en priorité ce qui a le plus d’impact utilisateur.

Cette posture est plus solide qu’un discours de conformité globale non démontrée.

## Ce que cette démarche apporte

Une démarche continue bien menée permet :

- d’améliorer progressivement la qualité perçue ;
- de réduire les incohérences ;
- de rendre l’application plus agréable à utiliser ;
- de limiter certaines erreurs d’usage ;
- de rapprocher le produit d’un niveau d’accessibilité plus sérieux.

## À retenir

Dans ONY, l’accessibilité et la qualité doivent être pensées comme un travail continu.

La bonne logique consiste à :

- corriger progressivement ;
- intégrer les bons principes dans le cycle projet ;
- documenter les décisions ;
- améliorer en priorité ce qui a le plus d’impact.

Cette démarche rend l’amélioration réaliste, durable et compatible avec l’évolution normale du projet.
