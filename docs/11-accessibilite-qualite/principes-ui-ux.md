# Principes UI / UX

## Objectif de cette section

Cette page présente les principaux **principes UI / UX** à respecter dans **ONY** pour soutenir à la fois la qualité globale du produit et une meilleure accessibilité.

L’objectif est de formaliser des règles simples de conception et d’interface permettant de garder une expérience plus cohérente, plus lisible et plus fiable.

## Principe général

Une bonne interface ne repose pas seulement sur l’esthétique.

Elle doit aussi être :

- claire ;
- cohérente ;
- compréhensible ;
- prévisible ;
- utilisable dans des contextes variés.

Dans ONY, ces principes doivent soutenir une expérience orientée usage réel, en particulier sur mobile, avec une attention particulière à la simplicité d’accès aux actions importantes.

## Clarté visuelle

Les éléments importants doivent être immédiatement identifiables.

Cela implique notamment :

- une hiérarchie visuelle lisible ;
- des titres compréhensibles ;
- des blocs bien séparés ;
- des actions principales visuellement repérables ;
- une interface qui évite la surcharge.

La clarté visuelle améliore à la fois le confort d’usage et la qualité perçue.

## Cohérence

Une même action devrait fonctionner de manière similaire dans plusieurs endroits de l’application.

La cohérence doit se retrouver dans :

- les libellés ;
- les composants ;
- les positions d’actions ;
- les comportements d’interaction ;
- les codes visuels.

Plus une interface est cohérente, moins elle demande d’effort d’apprentissage.

## Lisibilité

La lisibilité concerne autant les textes que les interfaces.

Il faut veiller à :

- éviter les contrastes trop faibles ;
- utiliser une taille de texte suffisante ;
- conserver une bonne respiration entre les éléments ;
- limiter les zones denses ou confuses ;
- rendre les contenus faciles à parcourir.

La lisibilité est un point central pour l’accessibilité comme pour la qualité générale.

## Prévisibilité

L’utilisateur doit pouvoir anticiper ce qu’une action va produire.

Une interface de qualité évite :

- les comportements surprenants ;
- les changements de contexte mal annoncés ;
- les boutons ambigus ;
- les retours d’action insuffisants.

La prévisibilité réduit les erreurs et améliore la confiance dans l’application.

## Simplicité des parcours

Les parcours importants doivent rester lisibles et directs.

Cela implique de :

- limiter les étapes inutiles ;
- réduire la charge cognitive ;
- garder des actions principales clairement visibles ;
- éviter les écrans trop chargés ;
- faciliter les usages fréquents.

La simplicité ne signifie pas pauvreté fonctionnelle, mais meilleure hiérarchisation.

## Qualité mobile

Le contexte d’usage mobile doit rester une référence forte dans ONY.

Il faut donc veiller à :

- conserver des zones interactives accessibles ;
- éviter les interfaces trop serrées ;
- garantir une lecture correcte sur petit écran ;
- faciliter la navigation au pouce ;
- limiter les ruptures d’expérience entre écrans.

## États et retours utilisateurs

Une bonne interface doit indiquer clairement :

- ce qui est cliquable ;
- ce qui est actif ;
- ce qui est désactivé ;
- ce qui a réussi ;
- ce qui a échoué.

Les retours d’état sont essentiels pour éviter les ambiguïtés.

## Lien avec l’accessibilité

Ces principes UI / UX ne remplacent pas un référentiel d’accessibilité, mais ils contribuent directement à de meilleurs résultats.

Une interface plus claire, plus cohérente et plus lisible est généralement :

- plus facile à comprendre ;
- plus simple à utiliser ;
- moins génératrice d’erreurs ;
- plus proche d’une expérience accessible.

## À retenir

Les principes UI / UX à retenir pour ONY sont les suivants :

- clarté visuelle ;
- cohérence ;
- lisibilité ;
- prévisibilité ;
- simplicité des parcours ;
- qualité mobile ;
- retours d’état explicites.

Ils doivent servir de base commune pour guider les choix d’interface et améliorer progressivement l’expérience globale.
