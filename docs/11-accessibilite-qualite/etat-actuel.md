# État actuel

## Objectif de cette section

Cette page présente un **état actuel** de l’accessibilité et de la qualité dans **ONY**.

L’objectif est de poser une lecture honnête de la situation présente, afin de :

- situer le niveau de maturité du projet ;
- identifier les points déjà cohérents ;
- repérer les axes d’amélioration ;
- éviter une documentation trop théorique ou déconnectée du réel.

## Positionnement global

ONY est un projet déjà structuré, avec une logique produit réelle, une architecture technique définie et une documentation en cours de consolidation.

En revanche, sur le plan de l’accessibilité et de la qualité d’interface, la démarche reste encore en construction.

Cela signifie que le projet ne part pas de zéro, mais qu’il n’est pas non plus à un niveau de formalisation avancé sur tous les sujets.

## Points déjà favorables

Plusieurs éléments vont dans le bon sens.

### Volonté de cohérence

Le projet montre une volonté claire d’unifier les comportements, les parcours et la direction d’interface.

Cette intention est importante, car la qualité perçue repose beaucoup sur la cohérence générale du produit.

### Attention au mobile et à l’usage réel

L’application est pensée dans un contexte où l’usage mobile et l’usage terrain comptent fortement.

Cette contrainte pousse naturellement à rechercher :

- des interfaces lisibles ;
- des actions simples ;
- une navigation directe ;
- des composants compréhensibles.

### Documentation en cours de structuration

Le fait de documenter explicitement l’accessibilité et la qualité constitue déjà un signe positif.

Cela montre que ces sujets sont identifiés comme des enjeux réels et non comme des ajouts purement cosmétiques.

## Limites actuelles

L’état actuel reste néanmoins perfectible sur plusieurs aspects.

### Accessibilité non encore formalisée de bout en bout

À ce stade, l’accessibilité n’est pas encore documentée comme une conformité complète démontrée critère par critère.

Il existe probablement des points à améliorer sur :

- les contrastes ;
- la navigation clavier ;
- la clarté de certains composants ;
- la cohérence des états interactifs ;
- la hiérarchie visuelle ;
- certains comportements mobiles ou responsives.

### Hétérogénéité possible entre écrans

Comme dans beaucoup de projets en évolution, certaines zones de l’application peuvent être plus abouties que d’autres.

Cela peut produire :

- des écarts de style ;
- des comportements variables selon les écrans ;
- des niveaux de finition différents ;
- une expérience parfois moins homogène qu’attendu.

### Démarche qualité encore en consolidation

La qualité est bien présente comme objectif, mais toutes ses dimensions ne sont pas encore stabilisées au même niveau :

- accessibilité ;
- cohérence d’interface ;
- conventions UI ;
- critères de validation ;
- formalisation de la recette visuelle et fonctionnelle.

## Lecture honnête recommandée

La bonne manière de présenter l’état actuel est la suivante :

- le projet est sérieux et structuré ;
- des efforts de cohérence sont déjà visibles ;
- les bases sont présentes ;
- mais la démarche accessibilité et qualité reste encore en progression.

Cette posture est plus crédible qu’un discours trop affirmatif sur une qualité supposée parfaite.

## Priorités implicites

Au vu du projet, les priorités les plus logiques semblent être :

- homogénéiser les comportements UI ;
- améliorer progressivement les points d’accessibilité visibles ;
- documenter les bonnes pratiques retenues ;
- intégrer davantage ces sujets dans les validations.

## À retenir

L’état actuel d’ONY sur l’accessibilité et la qualité peut être résumé ainsi :

- une base sérieuse existe déjà ;
- la cohérence produit progresse ;
- la démarche est engagée ;
- mais elle n’est pas encore totalement formalisée ni homogène partout.

Cette section sert de point de départ réaliste pour la suite de l’amélioration continue.
